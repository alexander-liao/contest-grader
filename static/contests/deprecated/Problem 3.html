Problem 3: Convert Prefix to Postfix!

You have recently come across a page containing the magical formula of life. Unfortunately, it is written in prefix notation and your processor only reads postfix notation, so your task is to write a program that will convert from prefix notation to postfix notation.

In prefix notation, operators are written before their arguments. For example, + 2 3 means add 2 and 3 and - 2 3 means subtract 2 and 3 (2-3). For this problem, the only two operators will be + and *

In postfix notation, operators are written after their arguments. For example, 2 3 + means 2 and 3 added together and 2 3 - means 2 and 3 subtracted (2-3).

Your task is to convert an expression in prefix notation containing only +*1234567890 into the same expression in postfix notation. Two adjacent digits represent two different numbers. No spaces will be in the input or the output. To clarify, $ a b for some operator $ and arguments a and b translates to a b $.

Input Specifications

Input will consist of one line representing the prefix expression.

Output Specifications

Output will consist of one line representing the postfix expression.

Subtasks

For all points, the maximum depth of the expression parse tree is less than 100 and the maximum length of the expression is 108.
The time limit is 1 second.

Sample Input

+*+1234

Sample Output

12+3*4+

Sample Input

+*+12+34*+56+78

Sample Output

12+34+*56+78+*+
